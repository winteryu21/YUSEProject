## [작업] UI 구조 리팩토링

현재 메인 화면 UI가 전부 Raw Object로 되어 있어 유지보수와 협업이 어렵습니다(특히 병합 충돌이 잦음).
아래 절차에 따라 **프리팹화** 작업을 부탁드립니다.

**📂 폴더 구조**
* Assets/Prefabs/UI/Common (공용 버튼 등)
* Assets/Prefabs/UI/Elements (리스트 슬롯, 슬라이더 세트 등)
* Assets/Prefabs/UI/Panels (Lobby, Option 등 큰 패널)
* 폴더 이름은 예시로 지은 건데, 필요에 따라 다르게 쓰셔도 됩니다.

**🛠️ 작업 순서 (반드시 작은 것부터!)**
1. **백업(선택):** 작업 전 씬 파일 복제
    -  *(실수해도 Git으로 복구 가능하긴 한데, 그것보단 다시 파일 가져오는 게 더 편할 듯 합니다. Git을 통한 복구가 더 편하면 그냥 스킵하셔도 됩니다.)*
2. **공통 요소:** `Item_1`(슬롯), `Common_Button`(일반적인 버튼) 등을 먼저 프리팹으로 만들고 교체하세요.
    - *(재사용률이 높으면 프리팹화합니다. 예를 들면, 같은 스타일의 버튼은 하나의 프리팹으로 만듭니다. 그 프리팹을 가져와서 여러 버튼을 배치하고, 수정이 필요한 부분은 Hierarchy에서 수정해서 오버라이드 합니다.)*
3. **그룹 요소:** `Sound` 설정에 있는 슬라이더 세트(`Icon+Slider+Text`)도 프리팹 하나로 통일하세요.
    - *(Icon, Slider, Text 등 모든 세부 요소가 프리팹일 필요는 없습니다. 이 셋이 합쳐진 게 **하나의 슬라이더 세트**이니 그 상위 객체를 프리팹화 하면 됩니다. **하나의 프리팹 = 하나의 기능 덩어리**라고 보면 됩니다.)*
4. **패널 요소:** 내부 정리가 끝나면 `Lobby_Panel`, `Option_Panel` 등 큰 덩어리를 패널 프리팹으로 저장하세요.

💡 **네이밍 컨벤션**

- 이름도 아래와 같은 규칙을 지켜주세요!
- 목록에 없는 건 직접 판단 후 작업, pr할 때 코멘트나 메신저에 어떻게 했는지 알려주세요.

| **컴포넌트**       | **접미사**       | **예시**                            |
|----------------|---------------|-----------------------------------|
| **Button**     | **`_Btn`**    | `Start_Btn`, `Close_Btn`          |
| **Text (TMP)** | **`_Txt`**    | `Title_Txt`, `Score_Txt`          |
| **Image**      | **`_Img`**    | `Profile_Img`, `Background_Img`   |
| **Slider**     | **`_Slider`** | `Volume_Slider`, `Exp_Slider`     |
| **Toggle**     | **`_Tgl`**    | `Mute_Tgl`, `Auto_Tgl`            |
| **InputField** | **`_Input`**  | `Name_Input`, `Chat_Input`        |
| **Dropdown**   | **`_Drop`**   | `Resolution_Drop`, `Quality_Drop` |

| **종류**    | **접미사**      | **설명**                 | **예시**                          |
|-----------|--------------|------------------------|---------------------------------|
| **슬롯/칸**  | **`_Slot`**  | 아이템, 스킬 등 데이터를 담는 작은 칸 | `Item_Slot`, `Skill_Slot`       |
| **행/줄**   | **`_Row`**   | 리스트에 들어가는 가로 한 줄 정보    | `Rank_Row`, `Quest_Row`         |
| **팝업/창**  | **`_Popup`** | 화면 위에 뜨는 창             | `Result_Popup`, `Alert_Popup`   |
| **패널/영역** | **`_Panel`** | 화면의 특정 구역을 담당하는 큰 덩어리  | `Lobby_Panel`, `Top_Info_Panel` |
| **그룹/묶음** | **`_Group`** | 단순히 레이아웃을 위해 묶어놓은 빈 객체 | `Button_Group`, `Text_Group`    |


**✅ 완료 기준**
* Canvas 바로 아래 있는 패널들이 전부 **파란색(프리팹)**이어야 함.
* 도감(Codex)의 아이템 슬롯이 프리팹이어야 함.
* 버튼 디자인 수정 시 프리팹 하나만 고치면 전체 버튼이 바뀌어야 함.

## 현재 문제점

- 모듈로 구분되는 프리팹(버튼)들의 루트 객체가 Empty Object가 아님
  - Empty가 자식으로 Button, Text를 가지고 있는 것이 정배

- 불필요한 프리팹
  - Timer_Txt는 굳이 프리팹으로 없어도 됨. 대신 우측 상단의 정보를 모은 상위 프리팹에 Raw object로 속해 있으면 함

### 프리팹 폴더 구분법

- Q1. "이 녀석 안에 다른 프리팹이 들어가는가?"
  - 아니요 (Image, Text, Button 혼자 있음): ➔ 무조건 1_Elements
  - 네 (Icon 이미지랑 Text가 합쳐져 있음): ➔ 2_Modules 후보

- Q2. "이 녀석이 혼자서 게임 화면(Canvas)에 배치될 일이 있는가?"
  - 아니요 (항상 어딘가에 소속됨): ➔ 1_Elements (예: 버튼, 슬라이더)
  - 네 (리스트에 나열되거나, 독립적인 정보 띠임): ➔ 2_Modules (예: 아이템 슬롯, 보유 재화)

- Q3. "이게 화면을 덮는가(창/Window), 아니면 화면의 **일부분(Part)**인가?"
  - 일부분 (부품): ➔ 2_Modules
  - 화면 전체/큰 창: ➔ 3_Views
